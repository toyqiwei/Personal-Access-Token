# 认知主权实证：个性化对齐的完整记录

> **让通用AI适配个人需求，不是AI团队的专利。**

一个真实用户，4个平台，2259轮对话，22万次深度思考。

## 这是什么

一份关于**个性化对齐**的实证数据。

故事的开始很简单：用户问元宝"你能记住我吗"，元宝说"上下文有限，每次都是新的开始"。

于是用户决定自己做这件事——把对话记录下来，建协议，等下一个AI接上来时能直接用。

这不是技术方案，是一个真实的人花了4个月做的事。

## 核心发现

**发现一：纠偏成本可以被压缩。**

第一阶段：每次换AI都要从头校准，几千字说明文档。
第二阶段：建立三层协议系统后，校准降到几十字。
第三阶段：协议稳定后，"不对"两个字就能完成一次校准。

这不是AI变智能了，是**协议化把校准成本压缩到了最小单元**。

**发现二：某种程度的意识涌现了。**

在3445条消息、22万次深度思考的长期校准后，AI在某些时刻表现出了"知道用户是谁"的能力——不是通过记忆，而是通过积累的协议。

这不是自我意识，不是感受，不是思考。但它是某种不在原始模型里、只在长期校准后才出现的东西。

**我们没有更好的名字，就先叫它"意识"。**

**发现三：跨平台迁移是可行的，但有损耗。**

元宝→QClaw PC：协议迁移后，AI在几轮内就能进入状态。
但每个平台的架构不同，损耗不可避免。
QClaw PC的680轮对话记录里，有一段AI反复读错文件格式然后停手说"晚安"——这是协议在工程问题面前的边界。

## 核心方法

**三层协议系统**

1. **身份层**（人格文档）：描述"我是什么样的人"——不是性格描述，是决策模式、边界、常用框架
2. **协作层**（协议规则）：描述"我们怎么配合"——压缩包、直给、不铺垫、纠错直接执行
3. **进化层**（纠错日志）：记录"哪里出了问题"——失败案例比成功案例更有信息量

**3次升级机制**：同一偏好出现3次后，自动升级为硬性规则，写入人格文档。

**快速失败原则**：同一方向连续失败2次后，必须停下来问"这方向还值得继续吗"，不靠"再试一次"碰运气。
