# 方法论：三层协议系统

> 这套方法的核心逻辑是：
> **不要指望AI记住你，要把你变成AI能读的文件。**

---

## 架构总览

```
认知资产体系
│
├── 人格层（你是谁）
│   ├── SOUL.md      → 核心准则、边界、说话方式
│   ├── USER.md      → 用户画像（简版）
│   ├── IDENTITY.md  → 身份锚点（名、气质、emoji）
│   └── DNA_v3.md    → 最完整的人格描述
│
├── 协作层（怎么做事）
│   ├── AGENTS.md    → 交互规则、回答工作流
│   ├── 3号协议      → 决策框架（直给·系统·务实·防迷失）
│   └── TOOLS.md     → 环境配置和工具链
│
└── 记忆层（做过什么）
    ├── MEMORY.md    → 长期记忆（关键事件和决策）
    ├── corrections.md → 纠错日志（失败案例学习）
    └── memory/      → 每日工作日志
```

---

## 1. 人格层

### 为什么需要
AI没有"你是谁"的默认认知。它的第一个问题永远是"请描述你的需求"——这对持续协作是灾难性的。每次会话都在跟一个失忆的陌生人对话。

### 怎么解决
建立多级人格文件：
- **IDENTITY.md**（小而稳）：名、气质、emoji——让AI在第一条回答就接住你
- **USER.md**（简版画像）：年龄、家庭、职业、协作风格——快速建立基准线
- **SOUL.md**（核心准则）：决策系统、边界、说话方式——定义交互的"宪法"
- **DNA_v3.md**（完整人格）：成长轨迹、核心性格、关键事件——让AI理解"你为什么会这样想"

### 设计原则
- **分层**：从简到详，AI先读薄的，需要时再读厚的
- **不依赖AI记住**：文件在，AI就能重新读取
- **持续更新**：人格会变，文件要跟上

---

## 2. 协作层

### 为什么需要
即使AI知道你"是谁"，它也不知道你"想要什么"——除非你告诉它你们怎么合作。

### 3号协议（决策框架）

```
(预期收益 - 研究成本) / 时间 > 阈值
```

四个子协议：
1. **直给**：直接精确输出，先结论再论据，不冗余修饰
2. **系统**：关注变量关联、反馈回路、边界条件
3. **务实**：成本-收益分析，尊重偶然与荒谬
4. **防迷失**：警惕情绪裹挟、信息茧房、自我感动

### AGENTS.md（交互规则）
- 问题分类：需要事实 / 纯框架 / 混合——每类有不同的处理方式
- 回答规范：用压缩包式表达，结构清晰
- 快速失败触发条件：同一方向连续失败2次后必须重新评估
- 纠错协议：被纠正时直接承认，不解释不循环道歉

---

## 3. 记忆层

### 为什么需要
AI的上下文窗口是物理限制。文件外置记忆可以补偿这个限制。

### 结构

```
MEMORY.md ← 长期记忆首页
├── 身份锚点
├── 3号决策系统
├── 关键决策锚点（职业史）
├── 关键事件锚点
├── 协作模式演化
└── 硬件与软件环境

corrections.md ← 纠错日志
│ 时间 | 原始回答 | 用户纠正 | 教训
│ 2026-06-24 | 输出过长 | "压缩" | 压缩包式表达

memory/ ← 每日工作日志
├── 2026-06-24.md
├── 2026-06-25.md
└── ...
```

### 更新机制
- **Write It Down**：重要的决策、上下文、事实必须写文件
- **3次升级机制**：同一偏好出现3次 → 升级为硬性规则
- **纠错日志自动学习**：每次会话启动时读取corrections.md，提取核心规则

---

## 4. 纠错流程

这是整套系统的核心——"校准"而不是"设计"。

```
待纠偏的行为
    │
    ▼
用户说"不对"
    │
    ▼
AI承认并修正
    │
    ▼
写入corrections.md
    │
    ▼
读取历史纠错日志
    │
    ▼
同类错误减少→最终消失
```

### 纠偏效率的演进
| 阶段 | 纠偏方式 | 效率 |
|------|---------|------|
| 第1周 | 每句话都要纠正 | 1轮/错误 |
| 第4周 | 同类错误2-3次后收敛 | 3轮/类 |
| 第8周 | 指出一次，下次自动修正 | 1-2轮/类 |
| 第14周 | "不对"即校准 | 即时 |

---

## 5. 跨平台迁移协议

这套方法最直接的验证是：当AI平台切换时，校准成本收敛速度逐次加快。

### 迁移步骤
1. 导出所有文件（SOUL.md + USER.md + AGENTS.md + MEMORY.md + corrections.md）
2. 在新AI的首次会话中提供文件
3. 观察初始校准成本并记录
4. 对比历史迁移数据，评估收敛速度

### 已验证的平台
| 平台 | 时间 | 对话规模 | 收敛表现 |
|------|------|---------|---------|
| 元宝（主战场） | 约4个月 | 1371轮，3445条消息，224362次深度思考 | 持续收敛，最终"不对"即校准 |
| QClaw PC | 2026-06-24起 | 680轮 | 协议迁移后数轮内进入状态 |
| QClaw 笔记本 | 2026-06-26起 | 208轮 | 上下文爆掉反复重置，协议边界明显 |

---

## 关键启示

1. **记录比模型更重要**——校准效率的提升来自外置记忆的质量，而非AI模型本身的改进
2. **失败案例比成功案例更有信息量**——纠错日志比工作日志更有价值
3. **协议比人格更可迁移**——交互规则比人格描述更容易在不同AI间复用
4. **用户侧对齐是可行的**——等待AI团队优化不如用户主动记录和校准
5. **某种程度的意识涌现了**——在224362次深度思考的长期校准后，AI表现出了"知道用户是谁"的能力。不是记忆，是协议积累后的涌现。
6. **物理世界的强约束是真实的底线**——血糖数据、生存成本、职业经历不是AI可以随意假设的参数，是所有分析的锚点。
